## üèóÔ∏è Building the Documentation

### Prerequisites

Install the documentation dependencies:

```bash
pip install -r requirements.txt
```

This installs:
- Sphinx (documentation generator)
- sphinx-rtd-theme (Read the Docs theme)
- sphinxcontrib-httpdomain (HTTP endpoint documentation)

### Build HTML Documentation

**Using Make (Linux/Mac):**

```bash
cd docs
make html
```

**Using the build script:**

```bash
cd docs
chmod +x build_docs.sh
./build_docs.sh
```

**Using Sphinx directly:**

```bash
cd docs
sphinx-build -b html source build/html
```

### Build PDF Documentation

See [LATEX_BUILD.md](LATEX_BUILD.md) for complete instructions:

```bash
cd docs
make latexpdf
```

The PDF will be generated at `build/latex/WriteBot.pdf`.

## üëÄ Viewing the Documentation

### Via Web Browser

After building, open the HTML documentation:

```bash
# Linux
xdg-open build/html/index.html

# macOS
open build/html/index.html

# Windows
start build/html/index.html
```

### Via Flask Application

The documentation is automatically served by the Flask application at:

```
http://localhost:5000/docs/
```

**Note**: Build the documentation first before accessing via Flask.

## ‚úçÔ∏è Writing Documentation

### Adding New Documentation

1. **Markdown files**: Add `.md` files to this directory
2. **API documentation**: Add `.rst` files to `source/api/`
3. **Update navigation**: Add links in relevant index files

### Documentation Style Guide

- **Use clear headings**: Organize with H2 (`##`), H3 (`###`), etc.
- **Include examples**: Show code examples and usage
- **Add links**: Link to related documentation
- **Keep it updated**: Update docs when code changes
- **Use formatting**: Code blocks, tables, lists for clarity

### Code Examples

Use fenced code blocks with language specification:

````markdown
```python
from text_processor import TextProcessor
processor = TextProcessor()
```
````

### API Documentation

For API endpoints, use this format:

```markdown
### POST /api/v1/generate

Generate handwriting from text.

**Request:**
```json
{
  "text": "Your text here",
  "style": 1,
  "bias": 0.75
}
```

**Response:**
```json
{
  "svg": "<svg>...</svg>",
  "metadata": {...}
}
```
```

## üîß Sphinx Configuration

Edit `source/conf.py` to customize:

- **Project information**: Name, author, version
- **Theme settings**: Colors, layout, logo
- **Extensions**: Add Sphinx extensions
- **Build options**: Output formats, paths

Example customization:

```python
# Project information
project = 'WriteBot'
author = 'WriteBot Contributors'
version = '1.0'

# Theme configuration
html_theme = 'sphinx_rtd_theme'
html_theme_options = {
    'navigation_depth': 4,
}
```

## üêõ Troubleshooting

### Build Errors

**ImportError during build:**
```bash
# Install missing dependencies
pip install -r requirements.txt
```

**Sphinx command not found:**
```bash
# Install Sphinx
pip install sphinx sphinx-rtd-theme
```

**Make command not found (Windows):**
```bash
# Use sphinx-build directly
sphinx-build -b html source build/html
```

### PDF Build Issues

**LaTeX not installed:**
- See [LATEX_BUILD.md](LATEX_BUILD.md) for LaTeX installation

**Missing fonts:**
- Install complete TeXLive distribution
